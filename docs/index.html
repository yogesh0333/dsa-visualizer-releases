Having trouble opening the app?

macOS: “DSA Visualizer” is damaged and can’t be opened

This usually means the app is unsigned (not notarized by Apple). The download is not corrupted — Gatekeeper is blocking it.

Option A — Open once:

Copy DSA Visualizer.app from the .dmg into Applications. Then right‑click the app → Open → confirm Open.

Option B — Remove quarantine (Terminal):

xattr -cr "/Applications/DSA Visualizer.app"

Adjust the path if the app lives elsewhere. Then try opening the app again. A future signed & notarized build won’t need this (requires an Apple Developer account).

Linux: install .deb or run .AppImage

CI builds are amd64 for Debian/Ubuntu-style systems. Download the Linux column assets from the table below (or use the green “Download for Linux” button when present).

.deb (install with dependencies):

sudo apt install ./dsa-visualizer_*.deb

If that fails:

sudo dpkg -i ./your-file.deb
sudo apt-get install -f

AppImage (portable):

chmod +x ./DSA*.AppImage
./DSA*.AppImage
